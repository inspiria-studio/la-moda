# Send Authentication Token Tool

## Description
Tool responsible for initiating user authentication process by sending an access token via email. This is the first step of the VTEX two-step authentication process.

## Functionality
This tool performs the authentication token sending process via email, executing two steps:
1. Obtaining the authentication token from the VTEX system
2. Sending the access token to the user's email

## Parameters

### Input
- **email** (string, required): User email address to which the token will be sent
  - Example: `"user@company.com"`
  - Contact field: `true`

### Required Credentials
- **BASE_URL**: VTEX store base URL (ex: `https://weni.vtexcommercestable.com.br`)
- **LOCALE**: Localization for authentication process (ex: `pt-BR`)

## Response

The tool returns information about the token sending process:

```json
{
  "result": {
    "authStatus": "Success",
    "token": "abc123def456"
  },
  "email": "user@company.com",
  "auth_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
}
```

### Response Fields
- **result**: Token sending result
  - **authStatus**: Operation status (Success/Error)
  - **token**: Token sent by email (if applicable)
- **email**: Email to which the token was sent
- **auth_token**: Internal authentication token for subsequent validation

## Internal Process

1. **Account Name Extraction**: Extracts account name from BASE_URL
2. **Authentication Token Retrieval**: Makes call to obtain authentication token
3. **Access Key Sending**: Sends access token to user's email
4. **Data Return**: Provides necessary information for next step

## APIs Used

### 1. Authentication Token Retrieval
**Endpoint:**
```
{BASE_URL}/api/vtexid/pub/authentication/start?scope={account_name}&locale={locale}
```

**Method:** GET

**Expected Response:**
```json
{
  "authenticationToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
}
```

### 2. Access Key Sending
**Endpoint:**
```
{BASE_URL}/api/vtexid/pub/authentication/accesskey/send
```

**Method:** POST (multipart/form-data)

**Headers:**
- `Accept`: application/json

**Data:**
- `authenticationToken`: Token obtained in previous step
- `email`: User email

## Use Cases
- B2B user login process initiation
- Access token sending via email
- Preparation for authentication validation

## Error Handling

### Invalid Email
```json
{
  "message": "Invalid email"
}
```

### Process Error
- Error logs are printed to console for debugging
- Returns generic "Invalid email" message to user

## Account Name Extraction
The tool automatically extracts the account name from BASE_URL:
- `https://weni.vtexcommercestable.com.br` → `weni`
- Removes protocol and takes first part before first dot

## Dependencies
- `requests` module for HTTP calls
- `re` module for regular expressions (account name extraction)

## Authentication Flow
1. **authentication** (this tool) → Sends token via email
2. **validate_token** → Validates token provided by user
3. **create_session** → Creates authenticated session (optional)

## Security
- Tokens are generated by VTEX system
- Process follows VTEX authentication standards
- Sensitive information is not exposed in logs
